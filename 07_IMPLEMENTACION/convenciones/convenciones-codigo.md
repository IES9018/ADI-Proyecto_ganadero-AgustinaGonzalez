# Convenciones de código – Ganadapp

## 1. Principios generales

- código legible antes que optimizado
- separación estricta de responsabilidades
- evitar lógica en controladores


## 2. Naming

- snake_case para Python
- nombres descriptivos en español técnico o inglés consistente
- entidades en singular


## 3. Estructura de commits

- feat: nuevas funcionalidades
- fix: correcciones
- docs: documentación
- refactor: reestructuración sin cambio funcional

## 4. Organización

- un archivo por entidad o caso de uso
- no mezclar dominio con infraestructura
