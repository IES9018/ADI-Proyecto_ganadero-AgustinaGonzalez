# Entrevista Inicial

## Nombre del proyecto

GanadApp

## Objetivo principal

Desarrollar un sistema de gestión ganadera para pequeños y medianos productores rurales de Malargüe, permitiendo administrar información productiva y sanitaria del ganado, mejorar la trazabilidad, controlar stock y generar reportes mediante una aplicación web y móvil.

## Problema a resolver

Actualmente gran parte de los productores rurales gestionan la información mediante registros manuales, dificultando:

- Seguimiento sanitario.
- Trazabilidad.
- Planificación productiva.
- Control de stock.
- Consulta de información histórica.
- Generación de reportes.

## Usuarios principales

- Productores ganaderos.
- Personal de campo.

## Funcionalidades principales

### Gestión de productores

- Datos personales.
- Ubicación.
- Contacto.
- Establecimientos asociados.

### Gestión de ganado

- Identificación individual.
- Especie.
- Raza.
- Edad.
- Estado sanitario.

### Trazabilidad

- Historial de movimientos.
- Eventos sanitarios.
- Cambios de estado.

### Vacunación y sanidad

- Calendario sanitario.
- Alertas.
- Tratamientos.
- Historial médico.

### Gestión de stock

- Alimentos.
- Medicamentos.
- Insumos.

### Administración de campos

- Parcelas.
- Capacidad.
- Distribución de animales.

### Reportes

- Sanitarios.
- Productivos.
- Estadísticos.

### Alertas

- Vacunación próxima.
- Controles pendientes.
- Stock crítico.

### Gestión económica básica

- Ingresos.
- Egresos.
- Gastos sanitarios.

## Restricciones

El sistema no deberá:

- Reemplazar sistemas oficiales.
- Integrarse con organismos externos.
- Incorporar inteligencia artificial.
- Utilizar dispositivos IoT.
- Implementar procesos financieros complejos.

## Plataforma

- Aplicación web.
- Aplicación móvil.

## Base de datos

PostgreSQL.

## Tiempo de desarrollo

Un cuatrimestre.
