# Requisitos del sistema

## 1. Requisitos funcionales

El sistema deberá permitir:

- registrar y gestionar animales individualmente
- registrar eventos sanitarios y productivos por animal
- consultar historial de trazabilidad completo
- gestionar stock de insumos básicos
- generar alertas simples por condiciones definidas (stock bajo, vacunación próxima)
- diferenciar usuarios por rol (productor y personal de campo)

## 2. Requisitos no funcionales

- sistema web accesible desde dispositivos estándar
- persistencia en base de datos relacional
- arquitectura modular o en capas
- diseño extensible para futuras funcionalidades
- funcionamiento en entornos con conectividad limitada
- interfaz simple y adecuada para usuarios rurales

## 3. Restricciones

- no integración con organismos externos
- no IoT
- no inteligencia artificial
- no módulo financiero complejo
- no arquitectura distribuida o en tiempo real
